### 람다함수
람다함수란 프로그래밍 언어에서 사용되는 개념으로 익명함수(Anonumous Functions)를 지칭하는 용어이다. 간단히 말하자면 함수를 보다 단순하게 표현하는 방법이라 볼 수 있다. 그렇다면 익명함수란 무엇인가? 말그대로 함수의 이름이 없는 함수이다. 익명함수들은 공통으로 일급객체라는 특징을 가지고 있는데, 일급 객체란 일반적으로 다른 객체들에 적용 가능한 연산을 모두 지원하는 개체를 가르린다. 함수를 값으로 사용할 수도 있으며, 파라미터로 전달 및 변수에 대입과 같은 연산들이 가능하다. 사실 일반적으로는 객체지향 언어보다 함수지향 언어에 가깝다.
```JAVA
// 기존 자바문법
new Thread(new Runnable() {
   @Override
   public void run() { 
      System.out.println("Welcome Heejin blog"); 
   }
}).start();

// 람다식 문법
new Thread(()->{
      System.out.println("Welcome Heejin blog");
}).start();
```

### 사용 이유/목적
람다 형태는 매개변수를 가진 코드 블록이지만, 런타임 시에는 익명 구현 객체(추상 메소드를 한개 포함한)를 생성한다. 병렬처리와 이벤트 지향 프로그래밍에 어울리며 딥러닝이나 빅데이터와 더불어 일종의 트렌드처럼 관심을 받고있다. 문법적으로 간결성을 가지고있어서 기존의 자바문법보다 쉽게 함수를 표현 할 수 있다.  

람다식은 결국 로컬 익명 구현객체를 생성하게 되지만, 이 람다식의 사용 목적은 인터페이스가 가지고 있는 메소드를 간편하게 즉흥적으로 구현해서 사용하는 것이 목적이다.

+ 기존 자바에서 interface를 이용해 다형성을 제공하기 위해서는 interface를 만들고, 그것을 구현한 class를 작성한뒤, 사용시에는 interface타입의 참조변수에 interface를 구현한 class의 객체를 생성하여 사용했다.
  ```JAVA
  public interface Goods {
    public void doSome();
  }
  
  public class Computer implements Goods {
    @Override
    public void doSome() {
      System.out.println("do Operation!);
    }
  }
  
  public class Main {
    public static void main(String[] args) {
      Goods com = new Computer();
      com.doSome();
    }
  }
  
  // 1. 인터페이스 생성
  // 2. 인터페이스 구현 및 클래스 생성
  // 3. 인터페이스 타입의 참조변수에 인터페이스를 구현한 클래스 객체를 생성 및 대입하여 사용
  ```
+ 또는 실행하는 쪽에서 익명 객체를 만들어 사용했다. 하지만 Goods를 구현한 객체가 자주사용되어야 한다면, 아래 코드를 계속해서 반복해서 사용해야할 것이고 그렇게 되면 코드가 지저분해 진다.
  ```JAVA
  public class Main {
    public static void main(String[] args) {
      Goods com = new Goods() {
        public void doSome() {
          System.out.println("do Operation!");
          }
        };
        com.doSome();
      }
    }
  }
  ```
+ 하지만 람다식을 사용하는 경우 아래처럼 간결하게 표현할 수 있다.
  ```JAVA
  public class Main {
    public static void main(String[] args) {
      Goods com = () -> System.out.println("do Operation!");
      com.doSome();
    }
  }
  
  // 1. 인터페이스 생성
  // 2. 인터페이스 참조변수에 람다식을 대입하여 사용
  ```
  
### 람다의 장점
1. 코드의 간결성 : 불필요한 반목문의 삭제가 가능하고 복잡한 식을 단순하게 표현할 수 있다.
2. 지연연산 수행 : 지연연상을 수행함으로써 불필요한 연산을 최소화 한다.
3. 병렬처리 가능 : 멀티쓰레드를 활용하여 병렬처리를 사용 할 수 있다.

### 람다의 단점
1. 호출이 까다롭다(인터페이스에 선언된 메소드를 호출 해야함).
3. 람다 Stream 사용시 단순 for문이나 while문 을 사용하게되면 성능이 떨어진다.
4. 불필요하게 너무쓰면 오히려 가독성이 떨어진다.

### 표현식
람다식은 매개변수 + 실행문 으로 구성된다(접근자, 반환형 모두 생략), 모양은 ()->{}; 의 형태로 ()에 해당 인터페이스 함수의 매개변수를 입력하고 {}에 실핼할 코드를 작성한다.

1. 람다는 [매개변수 -> 함수몸체] 로 이용하여 사용 할 수 있다.
2. 매개변수가 하나일 경우, 타입이 모두 같을 경우 타입을 생략 할 수 있다. 
3. 함수몸체가 단일 실행문이면 괄호{}를 생략 할 수 있다.(함수몸체가 return문으로만 구성되어 있는 경우 괄호{}를 생략 할 수 없다.)
4. 매개변수가 1개이고 실행할 문장도 1개이면  (), {} 둘다 생략가능
```JAVA
// 사용할 인터페이스
public interface Calculator {
  public int cal(int num1, int num2);
}

// 1. 기본 사용
public static void main(String[] args) {
  Calculator cal = (int num, int num2) -> { return num1 + num2; };
  System.out.println(cal.cal(1 , 2));
}

// 2. 매개변수 타입 생략
public static void main(String[] args) {
  Calculator cal = (num1, num2) -> { return num1 + num2; };
  System.out.println(cal.cal(1 , 2));
}

// 3. 중괄호{} 생략
public static void main(String[] args) {
  Calculator cal = (num1, num2) -> num1 + num2;
  System.out.println(cal.cal(1 , 2));
}

// 4. (), {} 생략
public static void main(String[] args) {
  Calculator cal = num1 -> System.out.println(num1);
  cal.cal(1);
}
```

### 사용 조건
구현해야할 인터페이스의 추상메소드가 단 1개 이어야 한다. 2개 이상인 경우 어떤 메소드를 람다식으로 표현했는지 알 수 없기 때문!  
만약 람다식으로 사용하기 위한 인터페이스를 만들엇고, 잘 사용하다가 해당 인터페이스를 구현하는 클래스에서 또 다른 기능이 필요하게되어 해당 인터페이스에 메소드를 추가하면 에러가 발생한다. 왜냐하면 해당 인터페이스가 람다식을 위한 단일 추상메소드를 가진 인터페이스 이기 때문이다.  
그렇게 때문에 개발자가 모든것을 기억한 상태로 모든 인터페이스들을 검사하며 개발하기에는 한계가 있기에 @FunctionallInterface 어노테이션을 사용한다.(자세한 내용은 참조블로그 2)
