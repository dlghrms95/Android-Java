## Array vs List
배열은 데이터의 크기가 정해져 있고, 추가적인 삽입 삭제가 일어 나지 않으며 검색을 필요로 할 때 유리  
(데이터의 추가/삭제는 느리지만 조회가 빠름)  
리스트는 데이터의 크기가 정해져있지 않고, 삽입 삭제가 많이 일어나며, 검색이 적은 경우 유리  
(데이터의 추가/삭제는 빠르지만 조회가 느림)

### 배열(Array)
배열(Array)은 여러 데이터를 하나의 이름으로 그룹핑해서 관리 하기 위한 자료구조로 index와 값의 쌍으로 구성된다.  
index는 값에 대한 유일무이한 식별자이며, 논리적 저장 순서와 물리적 저장 순서가 일치하기 때문에 index로 해당 원소에 접근할 수 있다. (O(1))  
또한 연속된 메모리의 공간으로 이루어져 있으며, 배열은 정의와 동시에 길이를 지정하기 때문에 길이를 바꿀 수 없다.  
+ 장점  
인덱스를 통한 검색이 용이하고, 연속적이기 때문에 메모리 관리가 편하다.  
+ 단점  
크기가 고정되어 있기 때문에 어떤 엘리먼트가 삭제되면, 그 상태를 빈공간으로 남긴다.(메모리 낭비)  
정적이기 때문에 배열의 크기를 컴파일 이전에 정해주어야 하고 컴파일 이후 배열의 크기를 변동 할 수 없다.

### List
리스트는 배열과 다르게 빈 엘리먼트를 허용하지 않는다.(java에서는 일부 허용)  
또한 배열에서 인덱스가 값에대한 유일무이한 식별자 였다면 리스트에서 인덱스는 몇 번째 데이터인가의 의미를 가진다.  
불연속적으로 메모리공간을 차지하고 (링크)포인터를 통해 접근한다.  
+ 장점  
(링크)포인터를 통해 다음데이터의 위치를 가르키고있어서 삽입 삭제가 용이하며, 동적크기를 가진다. 또한 불연속적이므로 메모리 관리와 재사용성이 편리하다.
+ 단점  
검색 성능이 좋지 않으며, (링크)포인터를 통해 다음 데이터를 가르키므로 추가적인 메모리 공간이 발생
